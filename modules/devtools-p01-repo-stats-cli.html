01

Scaffold the module

Every Go project starts as a module. In your empty repo:

$ go mod init example/ghstat
$ touch main.go

That go.mod file is your project's identity and dependency ledger — the equivalent of package.json or requirements.txt, but built into the toolchain.

Verify
Run go run . now. Empty main.go won't compile yet — read the error. Getting comfortable with Go's compiler messages early pays off all module long.
02

Read the repo from the CLI

Before writing this, predict: with go run . golang/go, which index of os.Args holds "golang/go"? (Recall from L01: [0] is the program itself.)

Grill
Design decisions to defend before you code: what happens on ghstat with no argument? And on ghstat golang — no slash? Validate the owner/repo shape before making a network call: a request you can reject locally should never cost a round-trip.

    if len(os.Args) < 2 || !strings.Contains(os.Args[1], "/") {
        fmt.Fprintln(os.Stderr, "usage: ghstat <owner>/<repo>")
        os.Exit(1)   // non-zero = failure, the Unix way
    }

}
03

Call the GitHub API

The GitHub REST API is free for public data (60 requests/hour unauthenticated) — and it insists on a User-Agent header, which forces you one level deeper than http.Get: build a request, then Do it. The error idiom and defer from L01 still run the show.

func fetch(repo string) ([]byte, error) {
    req, err := http.NewRequest("GET", "https://api.github.com/repos/"+repo, nil)
    if err != nil {
        return nil, err
    }
    req.Header.Set("User-Agent", "ghstat-cli")   // GitHub requires this
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        return nil, fmt.Errorf("requesting repo: %w", err)
    }
    defer resp.Body.Close()   // runs on return — no leaks
    switch resp.StatusCode {
    case 200:  // fall through to read
    case 404:
        return nil, fmt.Errorf("repo %q not found", repo)
    case 403, 429:
        return nil, fmt.Errorf("rate limited — try again later")
    default:
        return nil, fmt.Errorf("GitHub returned %d", resp.StatusCode)
    }
    return io.ReadAll(resp.Body)
}
Grill
Why give 404 and 403 different error messages instead of one generic "request failed"? (Because the user's next move differs: fix the typo vs wait an hour. Error messages are UX — a lesson every CLI tool you'll ever build repeats.)
04

Decode JSON into a struct

This is the L01 payoff: struct tags. The API returns ~100 fields; you model only the five you need and tag them to match GitHub's snake_case keys.

type Repo struct {
    FullName   string `json:"full_name"`
    Desc       string `json:"description"`
    Stars      int    `json:"stargazers_count"`
    Forks      int    `json:"forks_count"`
    OpenIssues int    `json:"open_issues_count"`
}

var r Repo
if err := json.Unmarshal(data, &r); err != nil {
    return fmt.Errorf("decoding response: %w", err)
}
Verify
go run . golang/go should print six-digit stars. If a field comes back as its zero value, the struct tag doesn't match the JSON key — curl https://api.github.com/repos/golang/go | head -40 and compare key names character by character.
05

Present & ship

Format the output, exit non-zero on any error, then build a real binary.

$ go build -o ghstat .
$ ./ghstat golang/go
golang/go — The Go programming language
★ 129482 · forks 1803 · open issues 9312
$ echo $?      # 0 on success, 1 on failure
Ship checklist
gofmt -w . (canonical formatting) · go vet ./... (catches bugs) · try ./ghstat definitely/not-a-repo and confirm the not found message + non-zero exit, not a panic.
