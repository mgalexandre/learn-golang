Build an FX rates CLI in Go.

A hands-on build. You'll write every line yourself; this module gives the plan, the grilling, and the checkpoints. It applies concept module L01 · Foundations — same skills as the weather route, money-flavored.

🔨 Build module 📦 Applies L01 ⏱ ~60 min hands-on

What you'll ship

  • A command rates EUR USD that prints the live exchange rate
  • A real HTTP call, JSON decoded into a struct — including a map field
  • Honest error handling and a non-zero exit code on failure
  • A Go module you can go build into a single binary
01

Scaffold the module

Every Go project starts as a module. In your empty repo:

$ go mod init example/rates
$ touch main.go

That go.mod file is your project's identity and dependency ledger — the equivalent of package.json or requirements.txt, but built into the toolchain.

Verify
Run go run . now. Empty main.go won't compile yet — read the error. Getting comfortable with Go's compiler messages early pays off all module long.
02

Read the currency pair from the CLI

Before writing this, predict: with go run . EUR USD, which indexes of os.Args hold the two currencies? (Recall from L01: [0] is the program itself.)

Grill
Design decisions to defend before you code: what happens on rates with no arguments? And on rates eur usd in lowercase — reject it, or normalize with strings.ToUpper? Decide where input cleanup lives; that choice shapes your main.
func main() {
    if len(os.Args) < 3 {
        fmt.Fprintln(os.Stderr, "usage: rates <base> <quote>")
        os.Exit(1)   // non-zero = failure, the Unix way
    }
    base  := strings.ToUpper(os.Args[1])
    quote := strings.ToUpper(os.Args[2])
    // ... next step wires the fetch
}
03

Call the FX API

Use the free, key-less frankfurter.dev (ECB reference rates). The key learning here is the error idiom and defer from L01 — every I/O call returns an error you must handle.

func fetch(base, quote string) ([]byte, error) {
    url := fmt.Sprintf("https://api.frankfurter.dev/v1/latest?base=%s&symbols=%s", base, quote)
    resp, err := http.Get(url)
    if err != nil {
        return nil, fmt.Errorf("requesting rate: %w", err)
    }
    defer resp.Body.Close()   // runs on return — no leaks
    if resp.StatusCode != 200 {
        return nil, fmt.Errorf("FX API returned %d (unknown currency?)", resp.StatusCode)
    }
    return io.ReadAll(resp.Body)
}
Grill
Why return ([]byte, error) instead of printing inside fetch? (Answer: separation of concerns — fetch gets data, main decides how to present and how to fail. This is what makes it testable when this code grows into the P3 ledger.)
04

Decode JSON into a struct

This is the L01 payoff: struct tags — plus one twist the weather route doesn't have. The rates object has currency codes as keys, so you model it as a map, not a struct.

// {"base":"EUR","date":"2026-07-14","rates":{"USD":1.1721}}
type Quote struct {
    Base  string             `json:"base"`
    Date  string             `json:"date"`
    Rates map[string]float64 `json:"rates"`
}

var q Quote
if err := json.Unmarshal(data, &q); err != nil {
    return fmt.Errorf("decoding response: %w", err)
}
rate, ok := q.Rates[quote]   // comma-ok: was the key present?
Grill
Why is Rates a map[string]float64 instead of a struct with a USD field? (Because the keys are data, not schema — they change per request. Struct fields are for fixed shapes; maps are for dynamic keys.)
Verify
go run . EUR USD should print a plausible rate. If you get 0, either a struct tag doesn't match or you skipped the comma-ok check — dump the raw body temporarily and compare keys.
05

Present & ship

Format the output, exit non-zero on any error, then build a real binary. (Displaying with float64 is fine; storing and adding money as floats is not — the P03 ledger will grill you hard on that.)

$ go build -o rates .
$ ./rates EUR USD
1 EUR = 1.1721 USD (2026-07-14)
$ echo $?      # 0 on success, 1 on failure
Ship checklist
gofmt -w . (canonical formatting) · go vet ./... (catches bugs) · try ./rates EUR XYZ and confirm you get a clean error + non-zero exit, not a panic or a silent 0.
